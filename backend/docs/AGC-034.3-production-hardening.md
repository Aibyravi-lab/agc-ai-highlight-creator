# AGC-034.3 — Production Hardening (Phase 1)

## Overview

Five production protection mechanisms added to the AGC backend. No API contracts changed. No frontend changes.

---

## New Configuration Values

Added to `backend/app/config/config.py` (all overridable via `.env`):

| Variable | Default | Purpose |
|---|---|---|
| `MAX_UPLOAD_SIZE_MB` | `500` | Maximum upload size in megabytes |
| `MAX_CONCURRENT_JOBS_PER_USER` | `2` | Maximum active jobs per user |
| `TEMP_CLEANUP_HOURS` | `24` | Age threshold (hours) for old temp folder cleanup |

---

## Protections Implemented

### 1. Per-user Concurrent Job Limit

**Router:** `POST /pipeline/start`

Before creating a new job, the endpoint counts the user's jobs with status `pending` or `processing`. If the count is at or above `MAX_CONCURRENT_JOBS_PER_USER`, it returns:

```
HTTP 429 Too Many Requests
"Maximum concurrent jobs reached. Please wait for an existing job to finish."
```

Completed and failed jobs are not counted.

### 2. Upload Size Validation

**Router:** `POST /upload/`

File size is checked against `MAX_UPLOAD_SIZE_MB * 1024 * 1024` bytes before the file is written to disk. Oversized uploads are rejected with:

```
HTTP 413 Request Entity Too Large
"File exceeds the maximum upload size of 500 MB."
```

The error message reflects the configured limit dynamically.

### 3. Automatic Temporary File Cleanup

**Services:** `PipelineService`, `BackgroundJobService`, `CleanupService`

Temp files are cleaned up after every job — success or failure:

- **Extracted frames folder** (`storage/frames/{video_name}_{timestamp}/`) — deleted in a `finally` block inside `PipelineService._run_pipeline()`. Runs even if any pipeline step raises an exception.
- **Uploaded video file** (`storage/uploads/{uuid}_{name}.ext`) — deleted in a `finally` block inside `BackgroundJobService.run_pipeline()`. Runs after the pipeline completes or fails.
- **Age-based maintenance cleanup** (`CleanupService.cleanup()`) — moved to the `finally` block so it runs on both success and failure paths.

### 4. Old Temporary Folder Cleanup Utility

**Service:** `CleanupService.cleanup_old_temp_folders()`

Callable standalone utility that deletes items older than `TEMP_CLEANUP_HOURS` hours from:
- `storage/frames/` (frame extraction folders)
- `storage/thumbnails/` (thumbnail files)
- `storage/uploads/` (uploaded videos)

Does not delete final output files in `storage/highlights/` or `outputs/results/` (those are user deliverables).

Not scheduled automatically — call it manually or wire it to a scheduled task when needed.

## Manual Testing Checklist

### 1. Concurrent Job Limit

- [ ] Start one background job via `POST /pipeline/start`
- [ ] Immediately start a second job — it should succeed (1 running < 2 limit)
- [ ] Immediately start a third job — should receive **HTTP 429** with the message `"Maximum concurrent jobs reached. Please wait for an existing job to finish."`
- [ ] Wait for jobs to complete, then confirm a new job can be started again
- [ ] Confirm completed/failed jobs are not counted toward the limit

### 2. Upload Size Validation

- [ ] Upload a file under 500 MB — should succeed with HTTP 200
- [ ] Upload a file exactly at 500 MB — should succeed
- [ ] Upload a file over 500 MB — should receive **HTTP 413** with size-limit message
- [ ] Confirm the response message reflects the configured limit (not hardcoded)
- [ ] Set `MAX_UPLOAD_SIZE_MB=10` in `.env`, restart server, upload a 15 MB file — confirm 413

### 3. Automatic Temporary File Cleanup (Success Path)

- [ ] Upload a video and start a pipeline job
- [ ] Wait for job to complete
- [ ] Confirm the uploaded file no longer exists in `storage/uploads/`
- [ ] Confirm the frames folder no longer exists in `storage/frames/`

### 4. Automatic Temporary File Cleanup (Failure Path)

- [ ] Start a pipeline job with an invalid or corrupt video path to trigger a failure
- [ ] Confirm the job status becomes `failed`
- [ ] Confirm the uploaded file is deleted from `storage/uploads/`
- [ ] Confirm no orphaned frame folders remain in `storage/frames/`

### 5. Old Temporary Folder Cleanup Utility

- [ ] Manually create an old test folder inside `storage/frames/` with an `mtime` older than 24 hours
- [ ] Call `CleanupService.cleanup_old_temp_folders()` in a test script or Python shell
- [ ] Confirm the old folder is deleted
- [ ] Confirm folders newer than 24 hours are not deleted
- [ ] Set `TEMP_CLEANUP_HOURS=1` in `.env` and confirm the cutoff changes accordingly

### 6. Regression

- [ ] Upload endpoint still accepts valid videos under the size limit
- [ ] Pipeline `/start` still returns a `job_id` and starts processing
- [ ] `GET /pipeline/job/{job_id}` still returns correct status and result
- [ ] `GET /pipeline/progress` still returns real-time progress during processing
- [ ] Final reel and highlight clips still exist in `storage/highlights/` after cleanup (outputs are not deleted)
